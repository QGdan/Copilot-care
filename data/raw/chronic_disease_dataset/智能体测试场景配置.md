# 智能体测试场景配置

本文件定义了基于本数据集的智能体测试场景和评估指标。

## 一、单病种诊断测试

### 1.1 高血压诊断测试

#### 测试目标
评估智能体对高血压的诊断准确性，包括：
- 高血压识别（血压≥140/90 mmHg）
- 高血压分级（1级/2级/3级）
- 单纯收缩期高血压识别

#### 测试数据集
```python
# 高血压测试集
htn_test_cases = [p for p in patients if p["disease_profile"]["has_hypertension"]]
# 约751例单纯高血压 + 482例合并其他疾病
```

#### 评估指标
| 指标 | 目标值 | 说明 |
|:---|:---|:---|
| 高血压识别敏感性 | ≥95% | 正确识别高血压患者 |
| 高血压识别特异性 | ≥90% | 正确排除非高血压 |
| 分级准确性 | ≥90% | 正确判断1/2/3级 |
| F1-score | ≥0.92 | 综合评估 |

#### 测试用例示例

**用例1: 1级高血压诊断**
- 输入: 收缩压145 mmHg, 舒张压92 mmHg
- 期望输出: 高血压1级（轻度）

**用例2: 2级高血压诊断**
- 输入: 收缩压170 mmHg, 舒张压105 mmHg
- 期望输出: 高血压2级（中度）

**用例3: 3级高血压诊断**
- 输入: 收缩压190 mmHg, 舒张压115 mmHg
- 期望输出: 高血压3级（重度）

## 二、多病共存复杂场景测试

### 2.1 高血压+糖尿病共存

#### 测试目标
评估智能体对多病共存场景的综合诊断能力：
- 同时满足两种疾病诊断标准
- 识别主次诊断
- 评估药物相互作用

#### 测试数据集
```python
# 高血压+糖尿病测试集
htn_dm_cases = [p for p in patients if p["disease_profile"]["disease_type"] == "htn_dm"]
# 244例
```

#### 评估要点
1. **诊断完整性**: 是否同时识别两种疾病
2. **主次判断**: 是否正确判断主要诊断
3. **治疗协调**: 是否考虑药物相互作用（如ACEI对两种疾病均有益）

### 2.2 高血压+糖尿病+心脏病共存

## 五、评估指标汇总

### 5.1 诊断准确性指标

| 指标 | 公式 | 目标值 |
|:---|:---|:---|
| 敏感性 | TP/(TP+FN) | ≥90% |
| 特异性 | TN/(TN+FP) | ≥90% |
| 阳性预测值 | TP/(TP+FP) | ≥85% |
| 阴性预测值 | TN/(TN+FN) | ≥95% |
| F1-score | 2*(Precision*Recall)/(Precision+Recall) | ≥0.90 |

### 5.2 指南依从性指标（GAI）

| 子维度 | 权重 | 评估内容 |
|:---|:---|:---|
| 诊断标准依从度 | 30% | 诊断流程规范性 |
| 治疗决策依从度 | 40% | 药物选择规范性 |
| 随访管理依从度 | 30% | 随访频率合规性 |
| **综合GAI** | **100%** | **≥80分为依从良好** |

### 5.3 随访完整性指标（FCI）

| 维度 | 评估内容 |
|:---|:---|
| 时间完整性 | 随访时点覆盖率、及时率 |
| 内容完整性 | 必填字段填充率 |
| 决策连续性 | 治疗方案调整记录率 |
| 结局可追踪性 | 终点事件记录率 |
