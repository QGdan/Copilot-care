# 后端执行三因子说明（前端展示口径）

## 一句话
治理看板前端只展示三类高价值执行因子，不展示抽象放行术语。

## 三类核心因子
1. 推理时延与重试热力图
- 目标：快速识别高负载阶段和超时风险。
- 展示项：阶段平均时延、重试次数、热度分级（稳定/偏高/高压）。

2. 分歧收敛曲线
- 目标：解释多 Agent 观点是否达到可交付阈值。
- 展示项：收敛指数、分歧跨度、当前协同状态。

3. 路由因果瀑布
- 目标：解释病例为何进入当前复杂度路径。
- 展示项：各因子贡献值、阶段阈值、最终路由决策。

## 展示原则
1. 不展示抽象流程术语，优先展示可解释的执行证据。
2. 每个因子都必须能映射到具体阶段与可追溯字段。
3. 颜色和状态只表达风险强弱，不表达“放行/阻断”语义。

## 使用场景
1. 评委演示：3 分钟内说明“系统如何判断、为何判断、当前是否稳定”。
2. 研发排查：定位异常是在时延、收敛还是路由因果层面。
3. 产品对齐：统一跨角色对系统状态的理解口径。
